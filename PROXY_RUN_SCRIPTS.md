# Proxy Run Scripts

This document describes the helper scripts added for launching many local `opera-proxy` instances from a CSV file and stopping them later.

## Files

- `run-proxies-from-csv.ps1`
- `run-proxies-from-csv.bat`
- `stop-opera-proxies.ps1`
- `stop-opera-proxies.bat`

## Purpose

These scripts help automate bulk local proxy startup from a CSV generated by:

- `-list-proxies-all-out`
- `-list-proxies-all`

Typical workflow:

1. generate a CSV with discovered endpoints
2. optionally filter by country and speed while generating the CSV
3. run `run-proxies-from-csv.ps1` or `.bat`
4. use `stop-opera-proxies.ps1` or `.bat` to stop all launched proxy processes

## `run-proxies-from-csv.ps1`

This script:

- reads a CSV file
- optionally filters rows by `speed_status`
- optionally filters rows by maximum `speed_ms`
- sorts rows before startup
- stops already running `opera-proxy` processes before launching new ones
- starts one local proxy process per CSV row
- assigns local ports sequentially starting from `8080` by default
- writes stdout/stderr logs for each launched process

### Required CSV columns

- `country_code`
- `ip_address`
- `port`

Optional speed-related columns:

- `speed_ms`
- `speed_status`

These become required if you use:

- `-OnlyOkSpeed`
- `-MaxSpeedMs`
- or if you want meaningful `SortBy speed`

### Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `-CsvPath` | string | `.\proxies.csv` | Path to CSV file with proxies |
| `-StartPort` | int | `8080` | First local port to bind |
| `-BinaryPath` | string | `.\bin\opera-proxy.windows-x64.exe` | Path to binary to start |
| `-BindAddress` | string | `127.0.0.1` | Local bind address |
| `-LogsDir` | string | `.\proxy-runs` | Base directory for per-run logs |
| `-OnlyOkSpeed` | switch | off | Start only rows with `speed_status=ok` |
| `-MaxSpeedMs` | int | none | Skip rows slower than this limit |
| `-SortBy` | string | `speed` | Sort rows by `speed`, `country`, or `ip` |
| `-NoStopExisting` | switch | off | Do not stop existing proxy processes before launch |
| `-ShowWindows` | switch | off | Show console windows for started proxy processes |

### Default behavior

By default the script first runs:

- `stop-opera-proxies.ps1`

This means a fresh launch batch will replace older `opera-proxy` processes started earlier.

If you want to keep existing instances alive, pass:

```powershell
-NoStopExisting
```

### Window behavior

By default started proxy processes are launched with hidden windows because:

- stdout is redirected to log files
- stderr is redirected to log files
- visible console windows would otherwise look empty

If you want to see individual proxy windows, pass:

```powershell
-ShowWindows
```

### Sort behavior

#### `-SortBy speed`

- rows with `speed_status=ok` and numeric `speed_ms` go first
- faster rows go before slower rows
- rows without valid speed data go later

#### `-SortBy country`

- sorted by `country_code`
- then by `ip_address`
- then by `port`

#### `-SortBy ip`

- sorted by `ip_address`
- then by `port`
- then by `country_code`

### Examples

Launch all rows from `proxies.csv` starting at local port `8080`:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run-proxies-from-csv.ps1
```

Same command through the batch wrapper:

```bat
run-proxies-from-csv.bat
```

Launch only successful speed checks up to `1500` ms:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run-proxies-from-csv.ps1 -CsvPath .\proxies.csv -OnlyOkSpeed -MaxSpeedMs 1500
```

Launch starting from port `9000` and sort by IP:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run-proxies-from-csv.ps1 -CsvPath .\proxies.csv -StartPort 9000 -SortBy ip
```

Launch without stopping already running proxy processes:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run-proxies-from-csv.ps1 -CsvPath .\proxies.csv -NoStopExisting
```

## `stop-opera-proxies.ps1`

This script:

- finds running `opera-proxy.windows-x64.exe` processes
- prefers exact path matches when `-BinaryPath` exists
- force-stops the matching processes

### Parameters

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `-BinaryPath` | string | `.\bin\opera-proxy.windows-x64.exe` | Preferred binary path to match |

### Examples

Stop all matching proxy processes:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\stop-opera-proxies.ps1
```

Stop using the batch wrapper:

```bat
stop-opera-proxies.bat
```

Stop processes for a specific binary path:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\stop-opera-proxies.ps1 -BinaryPath .\bin\opera-proxy.windows-x64.exe
```

## Log Files

Each `run-proxies-from-csv.ps1` execution creates a new timestamped directory under:

```text
proxy-runs\
```

Each launched proxy gets:

- one stdout log
- one stderr log

File names include:

- country code
- local port

## Important Nuances

- the run script starts one OS process per CSV row
- a large CSV can create many processes quickly
- local ports are assigned sequentially, so make sure the port range is free
- if `-MaxSpeedMs` is used, rows must have both `speed_ms` and `speed_status`
- if no rows remain after filtering, the script stops with an error
- by default, old proxy processes are stopped before new ones start

## Recommended Usage

Generate a CSV:

```powershell
.\bin\opera-proxy.windows-x64.exe -list-proxies-all-out proxies.csv -discover-repeat 5
```

Launch only reasonably fast proxies:

```powershell
run-proxies-from-csv.bat -CsvPath .\proxies.csv -OnlyOkSpeed -MaxSpeedMs 1200
```

Stop them all later:

```powershell
stop-opera-proxies.bat
```
